Reusable Workflows

7 workflows, one consistent pipeline

Call any workflow from your repo with uses: ITlusions/ITL.Github/.github/workflows/<name>.yml@main

_reusable-detect-version.yml
Detect Version
Resolves a semver version from the current branch and git tags. Outputs a raw semver, a Docker-safe image tag, and a PEP 440 version for Python wheels.
version image-tag python-version is-release
_reusable-ci-python.yml
CI — Python
Lint (ruff) → test (pytest with PostgreSQL service) → build wheel. Injects the resolved version into pyproject.toml before building. Uploads the wheel as a GitHub artifact.
ruff pytest postgres wheel
_reusable-ci-docker.yml
CI — Docker
Lint → test → Docker build, with support for a multi-repo checkout (repo + dependency side-by-side). Optionally pushes the image on main and develop.
ruff docker buildx multi-repo ghcr.io
_reusable-auto-tag.yml
Auto-tag
Reads the highest v* tag in the repo and pushes a patch bump tag (v0.1.3 → v0.1.4) to the given commit SHA. Runs on main only.
semver patch bump GH_PAT
_reusable-docker-retag.yml
Docker Retag
Promotes an existing image to a release tag using docker buildx imagetools create — a manifest-only operation. No rebuild, no digest mismatch between CI and production.
imagetools no rebuild promotion
_reusable-publish-pypi.yml
Publish PyPI
Downloads the wheel built by CI and publishes it to PyPI or TestPyPI via OIDC Trusted Publishing — no API tokens, no secrets. Routes stable tags to PyPI, pre-release tags to TestPyPI.
OIDC twine PyPI TestPyPI
_reusable-release-gh.yml
GitHub Release
Creates a GitHub Release for a given tag, auto-detects stable vs pre-release from the tag name, and optionally attaches a build artifact as a release asset.
release notes asset upload pre-release

Composite Actions

Step-level building blocks

Use in steps: with uses: ITlusions/ITL.Github/actions/<name>@main

Setup Python Environment
Installs Python, configures pip caching keyed on your requirements file, installs dependencies, and runs an optional extra install command.
uses: ITlusions/ITL.Github/actions/setup-python-env@main
Detect Release Type
Inspects a semver version string and outputs prerelease=true/false and release-type=stable/pre-release. Used internally by the publish workflow.
uses: ITlusions/ITL.Github/actions/detect-release-type@main
